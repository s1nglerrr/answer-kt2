# Тг-бот с оплатой и доставкой

## Введение

* Telegram-бот для локальной кофейни, созданный для увеличения клиентского потока в спокойные часы за счёт удобного оформления заказа, его оплаты и последующей доставки или самовывоза.

## Цели и задачи

### Цели:

Цель 1 - Создание Telegram-бота
Цель 2 - Организация онлайн-оплаты
Цель 3 - Обеспечение доставки заказов
Цель 4 - Возможность вызова курьера

### Задачи:

Задача 1 - Предоставить интуитивное меню с выбором нескольких напитков в одном заказе
Задача 2 - Реализовать оплату по QR-коду с передачей подтверждения об успешной транзакции
Задача 3 - Добавить выбор способа получения: самовывоз или доставка с указанием подробностей адреса
Задача 4 - В дальнейшем интегрировать чат на базе нейросети для взаимодействия с пользователями
Задача 5 - Внедрить систему NFT-стикеров для фиксации уникальных заказов и позиций меню

## Требования к функционалу

### Меню

1. Отображение доступных кофейных напитков
2. Возможность выбора нескольких товаров в рамках одного заказа

### Оплата

1. Генерация QR-кода для проведения платежа
2. Уведомление пользователя об успешном поступлении оплаты

### Получение заказа

1. Выбор формата получения: «самовывоз» или "доставка курьером"
2. При доставке - указание детального описания точки для курьера

### Пользовательское взаимодействие

1. Подключение развлекательного чат-бота на базе нейросетей
2. Интеграция NFT-стикеров в Telegram для персонализации заказов и позиций меню

## Требования к интерфейсу

### Простота и удобство

1. Понятный интерфейс с минимальным количеством шагов до оформления заказа
2. Чёткое представление меню и удобный выбор нескольких напитков

### Навигация

1. Кнопки и команды должны быть легко узнаваемыми и доступными
2. Простое перемещение между основными разделами: меню, оплата, получение

### Уведомления

1. Наглядные сообщения о статусе заказа и успешной оплате
2. Подсказки по выбору способа получения и деталям доставки

### Визуализация

1. Использование эмодзи для повышения вовлеченности пользователей
2. Отображение QR-кода в удобном и читаемом виде

## Нефункциональные требования

### Производительность

1. Быстрое реагирование бота
2. Обслуживание до 100 одновременных пользователей без заметной потери качества

### Безопасность

1. Защита персональных данных
2. Безопасная обработка платежей без риска утечки финансовой информации

### Поддержка и обслуживание

1. Простота обновления и техподдержки системы

## Интеграции и окружение

### Платформа бота

Использование Telegram Bot API для взаимодействия с пользователями

### Платежные системы

Интеграция с распространёнными платёжными сервисами для генерации QR-кодов

### База данных

Применение базы данных для хранения информации о клиентах, заказах и меню

### NFT-стикеры

Интеграция с платформой, поддерживающей генерацию и передачу NFT

### Нейросетевой чат

Подключение API для реализации интерактивного диалога с пользователями

## Ограничения и допущения

### Ограничения

1. Бот функционирует исключительно в рамках Telegram и использует только Telegram Bot API
2. Часть функций (например, внешние системы доставки или NFT-механики) будет внедряться поэтапно и может отсутствовать в первой версии

### Допущения

1. QR-код = СБП
2. Пользователи обладают базовыми навыками работы с Telegram
3. Регулярные обновления и техобслуживание не будут нарушать стабильность сервиса

## Критерии приёмки

* Все заявленные возможности реализованы и функционируют корректно
* Пользователь способен оформить заказ через бота: выбрать меню, оплатить, указать способ получения и получить подтверждение
* Интерфейс остаётся понятным и не вызывает затруднений
* Все кнопки и навигационные элементы работают как ожидается
* Платежные данные и конфиденциальная информация защищены
* Данные пользователей надёжно хранятся
* Интеграции со сторонними сервисами выполнены и протестированы
